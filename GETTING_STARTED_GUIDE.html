⚡ What is GymKiosk?

GymKiosk is a comprehensive gym management application with two complementary interfaces:

🖥️ Kiosk Mode (Desktop)
Fullscreen locked interface for gym floor use
📱 Mobile Companion
QR-code-based workout access on phones

💡 Key Capabilities

  • 135+ Exercises: Organized across 9 muscle groups
  • Intelligent Planning: Auto-generate workouts by difficulty & time
  • Nutrition Tracking: Meal plans integrated with workouts
  • QR Code Sharing: Instantly share workouts to phones
  • Workout Calendar: Track completed workouts with streaks
  • Exercise Form: Step-by-step instructions for every exercise
  • Personal Records: Weight & reps tracking
  • Daily Challenges: Gamified fitness goals

📦 System Requirements

For Kiosk Mode:

  • Windows 10/11 or macOS
  • Node.js 14+ (for server)
  • 4GB RAM minimum
  • 1920x1080 display recommended

For Mobile Access:

  • Same WiFi network as kiosk
  • Modern smartphone (iOS 12+ or Android 5+)
  • QR code reader app (built-in cameras have this)

🔧 Installation

Step 1: Prerequisites

Install Node.js from nodejs.org if not already installed.

Step 2: Extract Files

Extract the GymKioskApp folder to your desired location (e.g., Desktop).

Step 3: Install Dependencies

cd C:\Users\YourUsername\Desktop\GymKioskApp npm install
ℹ️ Note: This downloads required packages (Electron, Express, etc.). This may take 2-3 minutes.

📱 Mobile Companion Mode

Accessing Workouts on Phone

Method 1: QR Code Scan (Recommended)

  1. From kiosk, click "📲 Share" on any workout
  2. QR code appears on screen
  3. Open phone camera and point at QR code
  4. Tap notification to open workout
  5. Automatically loads on your phone's browser

Method 2: Manual URL Entry

  1. Find the Network IP (shown when app starts)
  2. On phone, open browser and type: http://192.168.x.x:3001/workout/[id]
  3. Workout loads automatically

🔧 Troubleshooting

Issue: App won't start

  • ✓ Ensure Node.js is installed: node -v
  • ✓ Run npm install first
  • ✓ Check no other app is using ports 3000-3001

Issue: QR code won't scan on phone

  • ✓ Phone and kiosk must be on same WiFi network
  • ✓ Check console shows Network IP (not localhost)
  • ✓ Visit http://localhost:3001/diagnostics to test
  • ✓ Allow Node.js through Windows Firewall

Issue: Workouts not saving to calendar

  • ✓ Ensure user is selected (check top-left)
  • ✓ Save button should say "💾 Save to Calendar"
  • ✓ Check browser console for errors (F12)
  • ✓ Try refreshing the page (F5)

Issue: Videos not playing

  • ✓ Check video files exist in assets/videos/
  • ✓ Video format should be .mp4
  • ✓ For missing videos, kiosk shows placeholder
  • ✓ Form instructions still available as text

Issue: Slow performance

  • ✓ Close unnecessary browser tabs/apps
  • ✓ Make sure only one instance of app is running
  • ✓ Check available RAM (4GB minimum recommended)
  • ✓ Update display drivers if available

Common Questions:

Q: Can I create custom exercises?

A: Currently no, but you can edit exercises.js file and restart the app.

Q: How do I backup my workouts?

A: Export calendar as JSON from settings, or download workouts.json from data/ folder.

Q: Can multiple phones access the same workout?

A: Yes! Share the QR code with multiple people or send them the URL directly.

Q: Is my data stored in the cloud?

A: No, everything is stored locally on the device except QR-shared workouts (stored on server).

Q: What happens if I close the app?

A: All your workouts and user data are saved. Just reopen the app to resume.
